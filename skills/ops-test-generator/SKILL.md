# OPS 测试用例生成器

## 触发词
"测试用例"、"测试计划"、"生成用例"、"测试准备"

## 流程
1. 输入 PRD/需求 → 提取模块/角色/路径/约束
2. 自动覆盖五场景: 正常流程 ≥3条, 边界 ≥3条, 异常 ≥2条, 权限 ≥2条, 兼容性 ≥2条
3. 输出结构化测试用例表 (编号/模块/场景/前置条件/步骤/预期/优先级)
4. 附加回归范围标注 + 风险关注点

## 模板
参见 docs/测试用例生成模板.md

## 创建 Issue
使用 .github/ISSUE_TEMPLATE/test_case.yml 模板创建事项
