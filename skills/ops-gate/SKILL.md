# OPS 质量门禁系统

## 触发条件
用户提到任何以下关键词时激活: "Bug"、"测试用例"、"验收"、"回归"、"复盘"、"质量"、"门禁"、"OPS"

## 六阶段流程

### 阶段一: 测试准备
- 基于 PRD / 需求说明 / 用户故事生成测试用例
- 覆盖: 正常流程、边界场景、异常场景、权限场景、兼容性场景
- 输出: 测试用例表、测试执行清单、风险关注点

### 阶段二: Bug 反馈
- 标准化 Bug 描述 (环境/账号/版本/复现步骤/截图/日志)
- 自动判定严重度 (P0/P1/P2/P3)
- 自动创建事项并关联需求/用例

### 阶段三: 研发定位
- 从日志/报错/代码变更中提取问题线索
- 整理排查路径和可能原因
- 修复后补充修复说明和影响范围

### 阶段四: 修复验证
- 根据修复说明生成回归测试清单
- 确认原问题 + 关联功能 + 边界场景
- 输出: 回归结果、遗留问题、风险点

### 阶段五: 测试验收
- 汇总测试通过/未解决Bug/阻塞项
- 输出: 验收结论、版本质量报告、上线风险清单

### 阶段六: 质量复盘
- Bug 分布/根因分类/修复效率分析
- 输出: 复盘报告、改进建议、规范沉淀
