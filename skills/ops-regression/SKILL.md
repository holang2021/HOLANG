# OPS 回归验证器

## 触发词
"回归"、"验证修复"、"修复好了"、"回归测试"

## 流程
1. 输入修复说明 → 自动生成三层回归清单
   - 第一层: 原Bug验证 (必须通过)
   - 第二层: 关联功能回归 (共用组件/API/数据流)
   - 第三层: 边界场景验证 (空值/极限值/并发)
2. 输出结构化回归结果 (通过/失败/阻塞/跳过)
3. 回归完成后更新Bug事项状态
