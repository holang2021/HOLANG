# OPS 版本验收器

## 触发词
"验收"、"上线检查"、"版本发布"、"能上线吗"

## 流程
1. 汇总: 测试通过项 / 未解决Bug / 阻塞项 / 遗留风险
2. 判定: 是否满足验收标准
3. 输出: 验收结论 + 版本质量报告 + 上线风险清单
4. 使用 .github/ISSUE_TEMPLATE/acceptance.yml 创建验收报告

## 验收结论
- 🟢 通过 — 可上线
- 🟡 有条件通过 — 部分修复后上线
- 🔴 不通过 — 延期上线
