# OPS 检查清单模板

## 使用时机
每次质量复盘后，将高频 Bug 类型转化为检查项。

---

## 检查清单：{版本号} {模块/阶段}

### 🔴 阻断项（不通过不能上线）

- [ ] {检查项 1}
- [ ] {检查项 2}

### 🟡 警告项（建议通过后上线）

- [ ] {检查项 1}
- [ ] {检查项 2}

### 🟢 信息项

- [ ] {检查项 1}

---

## 检查清单：代码提交前

- [ ] 变更行数 ≤ 100行（超限需拆分）
- [ ] 涉及文件 ≤ 5个（超限需评审）
- [ ] 含安全敏感代码（认证/授权/加密）→ 标注 security review
- [ ] 含数据库变更 → 附 migration + rollback 脚本
- [ ] 含 API 变更 → 附 API 契约文档更新

## 检查清单：Bug 修复后

- [ ] 含修复说明（改了什么、为什么、影响范围）
- [ ] 含回归测试（至少覆盖原 Bug + 关联功能）
- [ ] Beyoncé 规则：有测试证明此 Bug 不会重现
- [ ] 未引入新的 Console Error
- [ ] Performance 基线未劣化

---

> 检查清单随复盘报告同步更新，纳入 CI 门禁的 review-gate 阶段自动检查。
