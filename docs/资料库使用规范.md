# OPS 建设系统 — 资料库使用规范

## 目录结构

```
资料库根目录
├── 测试用例/        ← 阶段一产出：测试用例表、测试计划、风险清单
├── Bug报告/         ← 阶段二产出：标准 Bug 描述、截图、日志
├── 回归结果/        ← 阶段四产出：回归测试清单、回归结果、遗留问题
├── 验收报告/        ← 阶段五产出：验收结论、版本质量报告、上线风险清单
├── 复盘报告/        ← 阶段六产出：质量复盘报告、改进建议
└── 规范文档/        ← 沉淀产出：ADR、检查清单、编码规范
```

## 命名规范

| 文档类型 | 命名格式 | 示例 |
|---------|---------|------|
| 测试用例 | `{版本号}_{模块}_测试用例_{日期}.md` | `v1.2.0_用户模块_测试用例_20260615.md` |
| Bug 报告 | `BUG_{编号}_{模块}_{严重度}.md` | `BUG_001_登录_P1.md` |
| 回归结果 | `{版本号}_回归结果_{日期}.md` | `v1.2.0_回归结果_20260615.md` |
| 验收报告 | `{版本号}_验收报告_{日期}.md` | `v1.2.0_验收报告_20260615.md` |
| 复盘报告 | `{版本号}_质量复盘_{日期}.md` | `v1.2.0_质量复盘_20260615.md` |
| 规范文档 | `{编号}_{标题}.md` | `ADR-001_Git分支策略.md` |

## 版本号格式

```
v{major}.{minor}.{patch}
例: v1.2.0
```

## 责任人

| 阶段 | 上传人 | 审批人 |
|------|:---:|:---:|
| 测试用例 | 测试 | 产品 |
| Bug 报告 | 测试 | 研发 TL |
| 回归结果 | 测试 | 产品 |
| 验收报告 | 测试 TL | 产品 + 研发 TL |
| 复盘报告 | QA Lead | 全员 |
| 规范文档 | 起草人 | 技术委员会 |
